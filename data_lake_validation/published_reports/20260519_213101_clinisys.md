## 4. Root Cause Analysis

1. **Athena Large Table Truncation**:
   - **`view_extrato_atendimentos_central`**: Exactly 500,000 rows in AWS Athena versus 630,369 rows locally. This round number represents a hard cap in the production dlt/Glue ingestion settings.
   - **`view_medicamentos_prescricoes`**: 133,089 rows in AWS Athena versus 196,833 rows locally. This represents a lagging or truncated extraction batch in the AWS S3 pipeline.
2. **Accents and Special Characters (Encoding Issues)**:
   - Accented columns such as `bcf_embrião1_status` and `bcf_embrião1_fonte` are automatically sanitized in AWS Athena (non-ascii characters `ã` and `õ` replaced with `_`). Locally, DuckDB keeps the unicode columns exactly as they exist in MySQL.
